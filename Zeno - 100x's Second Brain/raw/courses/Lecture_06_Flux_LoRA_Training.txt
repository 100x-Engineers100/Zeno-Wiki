SECTION 1 — FLUX MODEL: WHY IT'S DIFFERENT FROM SDXL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SDXL model files are "zip files":
One checkpoint = model + CLIP + VAE packed together.
Load Checkpoint node unpacks all three automatically.

Flux model files are UNZIPPED:
You load each of the three components SEPARATELY.

The 3 nodes you need for Flux (replacing Load Checkpoint):

1. LOAD DIFFUSION MODEL
   - Select: "flux1-dev" (best model, ~24GB)
   - IMPORTANT: Change weight_type to "fp8_e4m3fn_fast"
     (This quantizes/compresses the model for GPU memory)
   - Quantization = compressing a model to fit in available VRAM

2. DUAL CLIP LOADER
   - Flux uses TWO clip models (SDXL used one)
   - Clip 1: "clip_l"
   - Clip 2: "t5xxl"
   - Set type and keep device as default

3. LOAD VAE
   - Select: "ae.safetensors" (Flux-specific VAE)
   - DO NOT use the standard SDXL VAE

Color-coded outputs match what Load Checkpoint output:
  Purple = model, Yellow = CLIP, Red = VAE

Additional Flux-specific node:
FLUX GUIDANCE node
- Placed BETWEEN positive CLIP prompt and KSampler
- Recommended guidance value: 3.5
- In KSampler: set CFG scale to 1 (not the usual 7)
  This is because Flux handles guidance differently internally.

SECTION 2 — WHAT IS A LoRA?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

LoRA = Low-Rank Adaptation. Fine-tuning technique for diffusion models.
You are not retraining the entire model (too expensive).
You train a small adapter that modifies the model's understanding
of a specific subject (a face, a product, an art style).

Why LoRA > IP Adapter for consistency:
- IP Adapter: image encoded as a prompt at inference time → approximate
- LoRA: model LEARNS the subject during training → exact recall
- After training, call the LoRA by trigger word in your prompt
  e.g., "photo of SRIDEV in a jungle" → model knows who SRIDEV is

Use cases:
- Face LoRA: consistent person across unlimited images
- Product LoRA: consistent product appearance (shape, label, color)
- Style LoRA: consistent visual style (e.g., Amul girl aesthetic)
- Brand LoRA: brand asset style replication

Sridev's live example:
Trained a LoRA on 40 Amul Girl comic images.
Output: new Amul-style comics generated with correct character, style,
even attempted fonts. Text wasn't perfect but aesthetic was preserved.
This demonstrated "brand style generator" use case.

SECTION 3 — LORA TRAINING SETUP (JavasLabs)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Platform: JavasLabs (GPU rental in browser)
Recommended GPU: A6000 or A5000 — use these specifically

Step-by-step (commands to be shared on Discord after session):

1. Open terminal in JavasLabs
2. Create a conda environment
3. Activate the conda environment
4. Install PyTorch (torch) — this is the deep learning framework
   needed for training. Also used for LLM fine-tuning later.
5. Clone the AI Toolkit repo (training framework)
6. Install pip requirements: pip install -r requirements.txt --break-system-packages
7. Create a .env file in the AI Toolkit folder
   - Name the file exactly ".env"
   - Add: HF_TOKEN=<your_huggingface_token>

Getting a HuggingFace token:
- Go to huggingface.co → Login → Profile → Access Tokens
- Click Create New Token → select "write" permissions
- Give it any name (e.g., "C7Live")
- Copy token IMMEDIATELY — it only shows once
- Paste into your .env file
- Save a local copy of the token

8. Log in to HuggingFace via terminal (paste token when prompted —
   nothing will visibly appear, that's normal)
9. Confirm "Login successful" + your username appears
10. Run the training command (to be shared)

SECTION 5 — DATASET PREPARATION (Image Collection)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Key principle: Good training = good data. Garbage in, garbage out.

For a face LoRA:
- Collect 15-25 photos of the person
- Include: different angles, different lighting, different expressions
- Include: some full-body, some close-up, some with accessories
- No heavy filters or edits
- Caption each image (AI toolkit auto-captions, or write manually)

For a product LoRA:
- Photograph product from multiple angles
- Different backgrounds, different lighting conditions
- Include the product label clearly visible in enough shots
- 15-20 images is sufficient sweet spot

SECTION 6 — FLUX ControlNets & Redux (IP Adapter)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Flux has its own ControlNet versions (not SDXL's):
- Available: Canny and Depth only (not Open Pose, MLSD, etc.)
- Loaded like LoRAs, not like SDXL ControlNets

Flux IP Adapter equivalent: "Redux"
- Works like IP Adapter but for Flux architecture
- Use Redux when you want image-based prompting with Flux
